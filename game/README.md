# LAST BUTTON 회색 박스 프로토타입

핵심 심리전 규칙을 검증하기 위한 Unity 오프라인 프로토타입입니다. 그래픽, 네트워크, 음성보다 다음 흐름이 이해되고 손에 잡히는지 먼저 확인합니다.

```text
협동 수리 → 키카드 탈취 → 공동 전력 전용 → 포드 충전 → 마지막 버튼
```

## 실행 환경

- Unity 6.5
- 기준 에디터: `6000.5.3f1`
- 별도 패키지 없음

## 실행 방법

1. Unity Hub에서 `game` 폴더를 프로젝트로 엽니다.
2. 빈 Scene을 연 상태로 Play를 누릅니다.
3. 런타임 부트스트랩이 테스트 공간을 자동으로 생성합니다.

## 조작

- `WASD`: 이동
- `Shift`: 질주
- 마우스: 시점
- `E` 길게 누르기: 수리 및 상호작용
- `F`: 키카드의 일회용 보안 교란기 사용
- `G`: 키카드 내려놓기
- `Q`: 바라보는 물체 밀치기
- `R`: 프로토타입 초기화
- `Esc`: 마우스 잠금 해제

## 현재 구현 범위

- 반복 가능한 수리 콘솔
- 수리·추격·배신을 수행하는 시뮬레이션 봇 3명
- 사람과 봇에게 적용되는 실제 넉백
- 배신자의 단거리 질주와 1회용 보안 교란기
- 기존보다 넓어진 구역형 테스트 맵
- 복구율 40%에서 활성화되는 물리 키카드
- 키카드를 직접 들고 내려놓는 상호작용
- 공동 복구율 10%를 사용하는 포드 충전
- 5초 동안 누르는 마지막 버튼
- 복구율 100%에서 가능한 공동 탈출
- 개인 탈출과 공동 탈출 결과
- 개발용 HUD와 사건 알림
- 6분 제한 시간과 공동·개인 예상 보상
- 물체 밀치기 기반 상호작용

## 의도적으로 제외한 범위

- 온라인 멀티플레이
- 인게임 음성
- 다른 플레이어와의 네트워크 밀치기 동기화
- 모듈형 맵
- 몬스터
- 메타 보상

포드 충전은 반복 테스트 속도를 위해 기획서의 20초보다 짧은 8초로 설정되어 있습니다.

`RAT` 봇은 복구율이 40%를 넘으면 키카드를 노리고, 포드를 충전해 마지막 버튼으로 도주합니다. `MISO`는 경보를 인식한 뒤 키카드 소유자를 추격하고 `BOLT`는 공동 수리를 계속합니다. 키카드 소유자는 `Shift` 질주와 `F` 보안 교란기를 이용해 추격을 따돌릴 수 있습니다. 봇은 물리 흐름 검증용이며 실제 사람의 거짓말과 음성 심리전을 대체하지 않습니다.

## 자동 검증과 Windows 빌드

배치 모드에서 다음 메서드를 실행할 수 있습니다.

- 상태 및 월드 검증: `LastButton.Prototype.Editor.PrototypeBatchValidation.Run`
- Windows 개발 빌드: `LastButton.Prototype.Editor.PrototypeWindowsBuild.Run`

Windows 빌드는 저장소의 `outputs/LAST_BUTTON_Prototype` 폴더에 생성되며 Git에는 포함되지 않습니다.
