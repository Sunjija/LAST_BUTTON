# LAST BUTTON 개발 현황 및 로드맵 v0.3

기준일: 2026-07-12

## 현재 게임 정의

4–6명이 고장 난 우주선을 함께 수리하지만, 플레이 도중 누구든 키카드를 탈취하고 공동 전력을 개인 포드로 돌려 혼자 탈출할 수 있는 5–7분 사회적 딜레마 게임이다. 고정 임포스터보다 자발적인 배신, 음성 대화, 물리적 저지와 마지막 버튼 앞의 난투를 핵심 장면으로 삼는다.

## 지금까지 완료된 작업

### 기획

- 3스테이지 세션과 스테이지당 5–7분 구조 수립
- 충성 플레이어의 공동 수리·공동 탈출 목표 정의
- 고정 임포스터가 아닌 플레이 중 자발적 배신 구조 채택
- 키카드 탈취부터 개인 포드 충전, 마지막 버튼까지 배신 전조 과정 설계
- 개인 탈출이 공동 복구율을 낮추는 손실 구조 반영
- 출시 이후 몬스터의 회피·유인·격리·역이용 방향 정리
- 2인 팀, AI 보조, 쇼츠 중심 홍보 전략을 전제로 범위 조정

### Unity 프로토타입

- Unity `6000.5.3f1` 프로젝트와 자동 생성 테스트 월드 구축
- 반복 가능한 수리 콘솔 3개와 공동 복구율 구현
- 복구율 40%에서 활성화되는 물리 키카드 구현
- 키카드 소지·내려놓기·피격 드롭 구현
- 공동 복구율 10%를 소비하는 개인 포드 충전 구현
- 5초간 유지해야 하는 마지막 버튼 구현
- 복구율 100%의 공동 탈출과 개인 탈출 결과 구현
- 실제 넉백과 물체 밀치기 구현
- Q 입력을 방향 고정 단거리 돌진으로 변경
- 돌진이 빗나가거나 벽에 충돌할 수 있고 실패 후딜레이가 발생하도록 구현
- 배신자의 질주와 1회용 보안 교란기 구현
- 수리 담당 BOLT, 보안 담당 MISO, 배신 담당 RAT 봇 구현
- 봇도 플레이어와 동일한 돌진 밀치기 규칙을 사용하도록 통일
- 격벽, 방, 문틈, 우회로와 탈출 구역 엄폐물이 있는 확장 맵 구현
- 복잡한 맵에서 봇이 이동할 수 있도록 격자 기반 경로 탐색 구현
- 6분 제한 시간, 예상 보상, 사건 알림과 봇 상태 HUD 구현
- 상태 검증, 월드 생성 검증과 Windows 개발 빌드 자동화

### 검증 결과

- Unity 컴파일 및 배치 검증 통과
- Windows 개발 빌드 생성 성공
- 자동 플레이에서 수리, 키카드 탈취, 포드 충전, 복도 이동과 단독 탈출까지 완료
- 런타임 예외 없음
- 봇은 규칙과 동선 검증에는 유효하지만 사람의 거짓말, 음성 심리전과 재미를 검증하지 못함

## DDD 검증 요약

현재 기획 골격은 7/10, 구현 체감은 4/10, 실제 사람 대상 검증 신뢰도는 3/10으로 판단한다.

### 작동하는 부분

- 수리율 상승과 다음 목표 해금이 진행 기대감을 만든다.
- 키카드 탈취, 전력 감소, 포드 경보와 추격이 단계적으로 긴장을 높인다.
- 빗나갈 수 있는 돌진은 명중, 회피와 역공의 짧은 변동을 만든다.
- 마지막 버튼을 5초 유지해야 하므로 성공 직전 저지되는 니어미스가 발생할 수 있다.
- 키카드 드롭은 소유권이 여러 번 바뀌는 난투를 만들 기반이 된다.

### 부족한 부분

- 보상 숫자를 사용할 곳이 없어 장기적인 기대감이 없다.
- 개인 탈출 보상이 위험과 사회적 비용에 비해 항상 매력적이지 않다.
- 맵, 목표 위치, 배신 시점과 사건이 고정되어 반복 예측 가능성이 높다.
- 실제 음성과 사람 플레이가 없어 사회적 도파민을 검증하지 못했다.
- 시각 효과, 사운드, 애니메이션과 충돌 피드백이 부족하다.
- 한 판 종료 후 다음 스테이지와 다음 판으로 이어지는 흐름이 없다.

## 앞으로 해야 할 작업

### 1. 그래픽 버티컬 슬라이스

- 프로젝트를 Built-in Render Pipeline에서 URP로 전환
- 구매한 모듈형 SF 에셋을 별도 브랜치에서 임포트하고 호환성 확인
- 포드실과 주변 복도 하나를 목표 품질로 완성
- 수리실, 보안실과 포드실을 조명과 색으로 구분
- 캐릭터 모델, 작업복 색상 변형과 Mixamo 애니메이션 적용
- 돌진 예고, 헛침, 충돌, 피격, 키카드 드롭과 버튼 작동 VFX 적용
- 발소리, 금속 충격, 경보, 포드 충전과 함선 배경음 적용
- 콜라이더, 라이트맵, 재질과 성능 검증

### 2. DDD 보상 루프 보완

- 공동 보상과 개인 보상의 의미 및 배분 방식 재설계
- 배신 성공 보상이 위험을 감수할 만큼 큰지 수치 조정
- 스테이지 사이 보상 소비처 구현
- 세션 내 장비, 일회성 도구 또는 외형 해금 중 하나를 최소 범위로 채택
- 완료, 실패, 탈취와 니어미스에 즉각적인 시청각 피드백 제공

### 3. 반복 다양성

- 수리 콘솔, 키카드, 충전기와 버튼 후보 위치를 여러 개 제작
- 판마다 목표 위치와 일부 통로 상태 변경
- 최소 3개의 맵 배치 변형 제작
- 정전, 문 잠김, 장비 고장 같은 소규모 사건 추가
- 배신 기회가 항상 같은 복구율에 열리지 않도록 조건 변형 검토

### 4. 세션 완성

- 한 스테이지 종료 후 결과 정산 화면 구현
- 자동 재시작과 다음 스테이지 전환 구현
- 3스테이지 누적 보상 및 최종 결과 구현
- 역할, 스폰과 목표 배치 랜덤화
- 플레이 로그 및 핵심 지표 기록

### 5. 실제 멀티플레이

- 4인 기준 방 생성과 참가 구현
- 플레이어 이동, 상호작용, 키카드와 밀치기 동기화
- 서버 권한 판정과 지연 상태에서 돌진 명중 검증
- 재접속과 호스트 이탈의 최소 예외 처리
- 근접 음성 또는 초기 테스트용 외부 음성 환경 준비

### 6. 사람 대상 플레이테스트

- 최소 4명으로 3판 연속 테스트
- 첫 배신 시점, 키카드 소유권 변경 횟수와 돌진 명중률 기록
- 배신 시도율, 배신 성공률과 공동 탈출률 기록
- 한 판의 강한 웃음·비명·언쟁 발생 횟수 기록
- 3판 후 한 판 더 플레이할 의향 확인
- 테스트 영상에서 쇼츠 후보 장면 추출

### 7. 후순위 확장

- 회피만 하는 몬스터가 아니라 유인, 격리와 역이용이 가능한 몬스터
- 추가 맵 테마와 환경 위험 요소
- 공개 매칭, 친구 초대와 로비 개선
- 외형 보상과 장기 진행 시스템
- Steam 페이지, 트레일러와 데모 배포 준비
