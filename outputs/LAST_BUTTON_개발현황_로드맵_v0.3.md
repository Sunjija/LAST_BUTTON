# LAST BUTTON 개발 현황 및 로드맵 v0.3

기준일: 2026-07-12

> 기획 기준 변경: 통합 게임 기획서 v0.19를 현재 기준으로 삼는다. 장기 리텐션보다 첫 8분 사건, 첫 60분 장면 밀도, 즉시 재시작, 7일 내 실제 재실행과 쇼츠의 상점 행동을 우선한다. 함께 탈출하는 데 필요한 황금 배터리, 방향 배신, 회수자의 재배신, 탈출 장치 준비와 원격 개입이 핵심이다. Unity 구현 현황은 v0.2이며 차이는 의도된 미구현 범위다.

## 현재 게임 정의

친구 4명이 우주선을 수리한 뒤 함께 탈출하는 데 꼭 필요한 황금 배터리를 운반한다. 운반자는 공동 배터리 소켓과 개인 탈출정 중 실제 방향으로 선택하고, 회수자도 새 도망자가 될 수 있다. 장치 담당자 세 명은 탈출 장치를 지키면서 자동문·냉각수·컨베이어·탈출정 방해 전파로 추격전에 개입한다. 4–6분 스테이지 4개와 원버튼 재시작으로 짧고 영상화 가능한 사건을 집중시킨다.

- 키카드 탈취부터 개인 탈출정 충전, 마지막 버튼까지 배신 전조 과정 설계

- 2인 팀, AI 보조, 쇼츠 중심 홍보 전략을 전제로 범위 조정

- 벽과 자동문, 방, 문틈, 우회로와 탈출 구역 엄폐물이 있는 확장 맵 구현

- 6분 제한 시간, 예상 보상, 사건 알림과 봇 상태 HUD 구현
- 상태 검증, 월드 생성 검증과 Windows 개발 빌드 자동화

### 검증 결과

- Windows 개발 빌드 생성 성공

## DDD 검증 요약

v0.19는 차별성·바이럴 문서 기준을 통과했지만 재미와 시장성은 검증되지 않았다. 다음 단계에서는 평점보다 첫 8분 사건, 30분 손바뀜, 장치 담당의 대기 응답, 실제 재시작과 쇼츠 UTM 행동의 폐기 기준을 우선한다.

### 작동하는 부분

- 수리율 상승과 다음 목표 해금이 진행 기대감을 만든다.
- 키카드 탈취, 전력 감소, 개인 탈출정 경보와 추격이 단계적으로 긴장을 높인다.
- 빗나갈 수 있는 돌진은 명중, 회피와 역공의 짧은 변동을 만든다.
- 마지막 버튼을 5초 유지해야 하므로 성공 직전 저지되는 니어미스가 발생할 수 있다.
- 키카드 드롭은 소유권이 여러 번 바뀌는 난투를 만들 기반이 된다.

### 부족한 부분

- 현재 프로토타입에는 함께 탈출하는 데 필요한 황금 배터리와 안전 잠금이 없어 욕심 전염을 검증하지 못했다.
- 현재 프로토타입에는 설비 예열, 스포트라이트 순환, 가변 계약금과 실패 손실이 없다.
- 맵, 목표 위치, 배신 시점과 사건이 고정되어 반복 예측 가능성이 높다.
- 실제 음성과 사람 플레이가 없어 사회적 도파민을 검증하지 못했다.
- 시각 효과, 사운드, 애니메이션과 충돌 피드백이 부족하다.
- 한 판 종료 후 다음 스테이지와 다음 판으로 이어지는 흐름이 없다.

- 반환과 재도주 사이의 고민 응답, 설비 이탈과 전원 호위 비율 기록

## 다음 마일스톤 완료 조건

`그래픽 버티컬 슬라이스 + v0.5 충성·계약 루프`를 다음 마일스톤으로 삼는다.

- 개인 탈출정 방과 주변 복도가 최종 아트 방향을 보여준다.
- 캐릭터가 걷고, 달리고, 돌진하고, 피격되는 모습을 확인할 수 있다.
- 버튼 앞 난투가 영상과 소리만으로 이해된다.
- 개인 탈출 보상과 공동 탈출 보상의 선택 이유가 명확하다.
- 계약자는 성공을 위해 최소 한 번 정상 업무로 위장하고 거짓말해야 한다.
- 충성 플레이어는 의심, 검사와 수리 중 무엇을 우선할지 선택해야 한다.
- 한 판 종료 후 다음 스테이지로 진행할 수 있다.

이후에 4인 네트워크 기능을 붙여 실제 사람 대상 DDD 검증으로 넘어간다.
