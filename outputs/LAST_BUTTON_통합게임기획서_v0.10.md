# LAST BUTTON — 통합 게임 기획서 v0.10

작성일: 2026-07-12  
문서 상태: 반복 검증안  
대상: 마이크를 사용하는 친구 4명  
플레이 시간: 5–7분 × 3스테이지

## 1. 최종 목표

각 플레이어는 3스테이지 종료 시 다음 두 조건을 모두 충족해야 고용을 유지함.

1. 개인 지분 `3.5B` 이상
2. 공동 발진으로 획득하는 안전 인증 `2개` 이상

여러 플레이어가 함께 성공할 수 있으며 단독 순위는 없음. 다른 플레이어의 성과를 낮춰도 자신의 조건은 쉬워지지 않음.

블랙 계약은 지분을 빠르게 채우는 역전 경로지만 안전 인증을 제공하지 않음. 계약 성공자도 남은 스테이지에서 공동 발진에 기여해야 최종 고용을 유지할 수 있음.

## 2. 보상과 인증

### 공동 성공

- 각 플레이어 지분 `+1B`
- 각 플레이어 안전 인증 `+1`
- 합법 업무와 개인 성과 최대 `+0.25B`

### 블랙 계약 성공

- 계약자 지분 `+2.5B`
- 다른 플레이어 지분 각각 `+0.75B`
- 아무도 안전 인증을 얻지 못함
- 개인 성과는 정상 지급

### 블랙 계약 실패 후 공동 성공

- 계약자 최종 스테이지 보너스 `-0.5B`
- 계약자는 공동 성공 `+1B`를 중복해서 받지 않음
- 계약자는 안전 인증 `+1` 획득
- 다른 플레이어는 `+1B`, 안전 인증 `+1`
- 개인 성과는 정상 지급

계약 실패의 `-0.5B`는 공동 성공 보너스를 대체하는 최종 스테이지 결과임. `1B - 0.5B`가 아님.

### 전체 실패

- 공동 보너스와 안전 인증 없음
- 개인 성과 절반 지급
- 전원 안전 위반 책임 `-0.25B`
- 계약 수락자는 총 `-0.75B`

## 3. 정상 충성 경로

```text
공동 성공 3회 = 3B / 안전 인증 3개
합법 업무·개인 성과 평균 합계 0.5B 이상
→ 3.5B와 인증 2개 충족
```

블랙 계약이 한 번 성공해 다른 플레이어가 `0.75B`만 받은 경우에도 나머지 두 공동 성공과 최대 `0.75B`의 성과를 합치면 `3.5B`, 인증 2개를 달성할 수 있음.

## 4. 계약 제안

- 네 명 모두 제안 대상
- 무작위 한 명에게 12초 비공개 제안
- 거절 시 다른 무작위 한 명에게 마지막 제안
- 모두 거절하면 계약자 없음
- 수락 15–25초 후 계약 존재 경보
- 제안 대상, 거절 여부, 지분과 인증 상태는 비공개
- 성공 `+2.5B`, 실패 시 최종 결과 `-0.5B`

계약 성공은 안전 인증을 주지 않으므로 한 세션에 계약만 반복해 최종 조건을 충족할 수 없음.

## 5. 합법 업무와 비계약 방해 억제

- 합법 업무는 공동 목표와 같은 방향의 비독점 조건만 사용
- 협동 수리와 완벽 판정은 참여자 모두에게 지급
- 마지막 입력, 희귀 자원 독점과 수리 지연 보상 없음
- 유형별 성과 상한으로 반복 파밍 방지
- 함선 실패는 자신의 지분과 안전 인증도 손해
- 발진 경보 전 고의 밀치기는 성과 차감

합법 업무 공개는 가능하지만 블랙 계약 보유 여부는 증명하지 않음. 공개 시 합법 업무 보너스 25% 감소.

## 6. 전원 위상 체크섬

네 명 모두 공동 발진 전에 참여함.

1. 각자 비공개 정상 기호를 받음
2. 각자 기호를 음성 또는 핑으로 보고함
3. 보고한 기호를 자기 패널에 입력함
4. 비계약자의 다른 입력은 즉시 오류
5. 계약자는 정상 기호와 계약 기호 중 선택
6. 계약 기호는 정상 성공처럼 처리
7. 계약자는 계약 기호를 정상 기호라고 거짓 보고해야 함

고정 역할이 없어 역할 배정과 협조 거부로 계약을 봉쇄할 수 없음.

계약자가 정상 기호를 입력하면 계약을 자진 포기함. 10초 뒤 `불법 전력 신호 소실` 방송이 발생하고 계약자는 실패 결과 `-0.5B`를 받음.

## 7. 두 후보 패널

계약 기호 입력 20–35초 후 다음 두 패널이 후보가 됨.

- 실제 계약 기호가 입력된 패널
- 다른 회로에서 센서 오작동으로 선택된 무고한 패널

공개 정보:

- 후보 두 명
- 두 명이 보고한 기호
- 입력 완료 속도: 빠름·보통·늦음

계약자는 자신이 정상 입력했다고 거짓 주장함. 무고한 후보는 실제 정상 입력을 주장함. 다른 두 명은 대화, 입력 당시 행동과 이전 동선을 판단함.

후보 지정 자체는 플레이어를 감금하거나 성과를 차감하지 않음.

## 8. 패널 재보정

- 두 후보 중 한 패널만 선택
- 두 명이 12초 작업
- 계약자 패널: 포드 전력 제거, 계약 실패
- 무고한 패널: 포드 전력 유지, 12초 손실
- 시스템은 계약자 이름을 공개하지 않음

재보정은 역할 처벌이 아니라 포드 전력 상태에 대한 한 번의 선택임.

## 9. 공동 발진과 개인 포드

### 공동 발진

- 원자로, 항법, 화물 고정 중 두 설비 작동 시 정상 진행
- 세 설비 작동 시 빠른 진행
- 한 설비 이하이면 감소
- 조작된 설비는 6초 자동 유지
- 계약자 이탈 후에도 두 명이 발진 유지, 한 명이 재보정 또는 추격 가능

### 개인 포드

- 포드 전력 유지 시 계약자에게 키카드 위치 표시
- 키카드는 물리적으로 회수·드롭·반환 가능
- 마지막 버튼 5초 유지 시 계약 성공
- 돌진은 방향 고정이며 빗나갈 수 있음

### 종료

- 공동 게이지 선행: 공동 성공
- 마지막 버튼 선행: 계약 성공
- 시간 초과: 전체 실패
- 성공 즉시 스테이지 종료, 관전 공백 없음

## 10. 코미디 수리

### 냉각수 호스

- 한 명이 호스를 잡고 한 명이 밸브를 잠금
- 놓치면 제한 범위 물줄기와 미끄러운 바닥
- 실패해도 진행 일부 유지

### 압력 밸브

- 한 명은 숫자 확인, 한 명은 밸브 조작
- 과압 시 1–3초 증기 충격과 물건 드롭
- 목표 구간 유지로 정상·완벽 판정

수리는 10–20초, 참여자 공동 보상, 짧고 회복 가능한 사고, 전체 초기화 없음의 원칙을 사용함.

## 11. DDD 리듬

```text
협동 수리와 물리 사고
→ 지분·안전 인증 진행
→ 합법 업무
→ 계약 유혹 또는 계약 경보
→ 체크섬의 거짓 기호
→ 실제 후보와 무고한 후보의 설명 대결
→ 한 번의 재보정 선택
→ 공동 발진과 키카드 추격
→ 마지막 버튼 니어미스
→ 계약자 공개와 고용 조건 정산
```

## 12. 초기 검증 범위

- 4인, 고정 맵 1개, 3스테이지
- 3.5B 지분과 안전 인증 2개
- 합법 업무 4종
- 블랙 계약 1종
- 전원 체크섬, 두 후보, 재보정 1회
- 냉각수 호스와 압력 밸브
- 2-of-3 공동 발진
- 키카드, 포드, 돌진과 마지막 버튼
- Discord 음성

6인, 공개 매칭, 자체 음성, 몬스터, 모듈 맵, CCTV, 리플레이, 전용 서버와 장기 능력 성장은 제외함.

## 13. 적대 검증 지표

- 계약 성공자가 남은 공동 발진을 방해할 합리적 이유가 없음
- 후보 선택이 시스템 정보만으로 자동 해결되지 않음
- 계약자가 거짓 기호와 거짓 설명을 사용한 판 90% 이상
- 무고한 후보의 불공정 체감 25% 미만
- 계약 수락률 20–65%, 성공률 30–50%
- 계약 없는 판 재플레이 의향 60% 이상
- 충성과 계약 재미 차이 1점 이내
- 3스테이지 재대결 의향 60% 이상

## 14. 폐기 기준

- 무고한 후보 지정이 반복적으로 임의 희생처럼 느껴짐
- 두 후보 선택이 사회적 판단이 아니라 동전 던지기로 느껴짐
- 계약자가 거짓 설명 없이 성공함
- 안전 인증 때문에 계약 선택이 사실상 불가능해짐
- 계약 성공자가 남은 스테이지를 방해함
- 계약 없는 판 수리가 재미없음
- 계약 성공이 트롤링으로 평가됨
- 3스테이지 후 재플레이 의향이 절반 미만임

## 15. 현재 상태

현재 Unity 프로토타입은 v0.2 규칙을 구현함. 반복 기획 검증 종료 전까지 v0.10은 코드에 반영하지 않음.
