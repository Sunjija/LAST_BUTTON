# LAST BUTTON 통합 게임 기획서 v0.19

작성일: 2026-07-12  
문서 상태: 단기 바이럴 전략 최종 기준안  
대상: 마이크를 사용하는 친구 4명  
플레이 시간: 스테이지당 4~6분, 총 4스테이지

## 1. 제품 전략

LAST BUTTON은 20시간 이상 유지하는 라이브 서비스가 아니라 첫날 2~5시간 동안 강한 사건을 반복 생성하고, 친구 호출로 몇 번 재실행되는 바이럴 파티 게임을 목표로 한다.

개발 우선순위:

1. 첫 8분 안의 공개 도주·강탈
2. 첫 30분 안에 네 명 모두 사건의 중심 경험
3. 한 시간당 실제 공유 가치가 있는 클립 3개 이상
4. 첫 세션 직후 자발적인 한 세션 더
5. 7일 안의 실제 그룹 재실행
6. 장기 메타와 콘텐츠 양

## 2. 한 줄 콘셉트

공동 발진에 필요한 황금 배터리를 원자로에 넣으면 모두 보상받지만 개인 탈출정으로 방향을 틀면 혼자 큰돈을 노릴 수 있고, 추격자가 배터리를 빼앗으면 그 욕심까지 이어받는 4인 협동 액션 게임.

```text
배신자 추리 X
공동 물건의 방향 선택·공개 추격·소유권 이전 O
```

## 3. 세션

- 4인 고정
- 4~6분 스테이지 4개
- 한 세션 16~24분
- 수리 목표 150초
- 배터리·발진 단계 60~90초
- 결과와 다음 장비 선택 20초 이내

한 세션 자체를 억지로 길게 만들지 않는다. 그룹이 결과를 확인한 뒤 같은 멤버로 즉시 다시 시작할 수 있는 원버튼 재시작과 역할·경로 재조합을 제공한다.

## 4. 대표 장면

```text
운반자가 탈출정으로 방향을 틂
→ 관제자들이 격벽·냉각수·재밍으로 원격 개입
→ 도망자가 돌진에 맞아 배터리를 떨어뜨림
→ 회수자가 원자로 대신 다시 탈출정으로 달림
→ 마지막 버튼 종료 0.2초 전에 배터리가 또 튕겨 나옴
```

영상의 중심은 회의와 정체 공개가 아니라 방향 전환, 충돌, 원격 사고와 배터리 손바뀜이다.

## 5. 스포트라이트와 관제 역할 순환

아직 다음 경험을 하지 못한 플레이어를 최초 운반자로 우선 지정한다.

- 원자로·탈출정 방향 선택
- 블랙 계약 자기장 강탈
- 떨어진 배터리를 주운 뒤 반환·재도주 선택

관제 역할도 세션 안에서 같은 역할을 두 번 맡기 전에 다른 관제를 우선한다. 한 플레이어가 같은 수동 역할에 반복 고정되지 않는다.

네 명 모두 핵심 경험을 마치면 최초 운반 횟수와 관제 중복이 가장 적은 조합을 선택한다.

## 6. 지분과 다음 스테이지 영향

- 공동 발진 `+1B`
- 수리 개인 성과 최대 `+0.5B`
- 기본 단독 탈출 `+2B`
- 뜨거운 배터리 단독 탈출 `+2.75B`
- 블랙 계약 성공 `+2.5B / +3B / +3.5B`
- 공개 도주·강탈 실패 `-0.5B`

지분 등급:

- 0~1.99B: C급
- 2~3.49B: B급
- 3.5~4.99B: A급
- 5B 이상: S급

누적 지분 최고 기록이 정수 단위를 처음 넘을 때마다 다음 스테이지에 한 장만 장착하는 일회용 보급 카드를 얻는다.

## 7. 뜨거운 배터리

처음 플레이하는 계정이 포함된 그룹의 첫 세션은 1스테이지 배터리를 뜨거운 배터리로 지정한다. 뜨거운 상태는 운반자가 배터리를 집을 때까지 비공개다.

- 개인 탈출 `+2.75B`
- 공동 발진 `+1B`
- 실패 `-0.5B`
- 운반자에게 1.5초 일회성 과열 가속
- 운반자에게 성공 시 급여 등급 변화 표시
- 다른 플레이어에게는 도주 선언 뒤 현상금 공개

뜨거운 배터리 스테이지에는 블랙 계약을 제안하지 않는다. 보상 충돌과 첫 판 규칙 과부하를 막는다.

같은 그룹의 두 번째 세션부터는 네 스테이지 중 하나를 완전 무작위로 지정한다. 첫 세션의 빠른 사건 발생을 보장하는 장치와 재실행의 예측 불가능성을 분리한다.

## 8. 블랙 계약

뜨거운 배터리가 아닌 스테이지에만 등장한다.

- 계약 미수락·단독 탈출 미성공 플레이어만 대상
- 12초 제안, 거절 시 다른 한 명에게 마지막 제안
- 수락 보증금 `0.25B`
- 성공 `2.5B / 3B / 3.5B`
- 포기 `-0.25B`
- 공개 실행 뒤 실패 총 `-0.5B`

운반자인 계약자는 2초 가속과 문 하나 4초 잠금을 얻는다. 비운반 계약자는 강탈 구역에서 1초 조준하는 자기장 강탈을 두 번까지 시도한다.

계약은 배터리를 생성하지 않는다. 배터리는 모든 스테이지의 공동 필수 물건이다.

## 9. 협동 수리

- 전력 릴레이: 떨어진 두 방의 박자 입력
- 냉각수 호스: 호스와 밸브, 꼬이면 물줄기·미끄러짐
- 압력 밸브: 숫자 전달과 과압 조절, 실패하면 증기 분출
- 에너지 셀: 무거운 셀 협동 운반

실패는 10~20초 안에 복구되고 전체 진행도를 초기화하지 않는다. 한 업무 반복으로 개인 성과 상한을 채울 수 없다.

첫 운반 방향 선택이 시작 후 6분을 넘으면 해당 세션은 바이럴 속도 게이트 실패로 기록한다.

## 10. 발진 예열

수리 완료 뒤 운반자·원자로·냉각수·통신 역할 위치를 3초 유지하면 30초 최종 단계가 시작된다.

- 세 관제자는 자기 설비를 8초 예열
- 2초 이상 이탈하면 복귀 후 5초 재부팅
- 운반자는 공동 소켓 또는 탈출정 경로 선택
- 공동 소켓 삽입과 세 설비 완전 예열 뒤 3초 점화
- 전원이 끝까지 호위하면 세 설비 재부팅 때문에 시간 내 공동 발진 불가

## 11. 관제자의 능동적 개입

각 관제자는 자리를 지키는 동안 추격 경로의 전술 화면과 전용 개입 1회를 사용한다. 자리를 떠나면 개입 권한을 잃고 직접 추격한다.

### 원자로 — 격벽 펄스

- 지정 격벽 4초 폐쇄
- 1초 전 바닥 경고
- 도망자·추격자 모두 차단 가능

### 냉각수 — 복도 분사

- 지정 구간 5초 미끄러움
- 1초 전 파란 경고
- 모든 플레이어와 바닥 배터리에 적용

### 통신 — 컨베이어 반전 또는 포드 재밍

- 한 스테이지에 둘 중 하나만 선택해 1회 사용
- 컨베이어 반전: 추격 복도의 이동 벨트 방향을 4초 반전, 모든 플레이어와 배터리에 적용
- 포드 재밍: 마지막 버튼 요구 시간 `+2초`, 입력 중 사용해도 기존 진행률 유지
- 통신 관제자는 초반 경로 개입과 후반 보험 중 선택

### 공용 과열 잠금

하나의 관제 효과가 유지되는 동안 다른 개입은 잠긴다. 효과가 끝난 뒤 1.5초가 지나야 다음 관제를 사용할 수 있다. 포드 재밍처럼 즉시 적용되는 효과는 발동 뒤 1.5초 잠금을 사용한다. 격벽·냉각수·컨베이어를 겹쳐 확정 저지할 수 없다.

관제 효과에 맞은 사람과 배터리 이동은 자동으로 쇼츠 로그에 기록한다.

## 12. 배터리 자기 잠금과 배신

도주 선언 전 배터리는 최초 운반자와 자기 결합한다. 몸은 밀리지만 배터리는 떨어지지 않는다.

- 공동 소켓 안전 통로: 상태 변화 없음
- 탈출정 경계선 3초 통과: 공개 도주 선언

선언 순간 자기 잠금이 풀리고 소지자 윤곽, 탈출정, 현상금이 공개된다.

공개 도주 뒤에는:

- 방향 고정 돌진 명중 시 배터리 드롭
- 벽·문·물체에 튕김
- 누구나 주울 수 있음
- 새 소지자가 탈출정 쪽으로 3초 이동하면 새 도망자
- 이전 도망자 `-0.5B`
- 공동 소켓 잠금 뒤 회수 불가
- 마지막 버튼 기본 5초 유지

단독 탈출 보상은 플레이어당 세션 1회다. 이미 성공한 플레이어는 배터리를 회수할 수 있지만 공동 소켓만 활성화된다.

## 13. 종료

### 공동 발진

- 도주하지 않은 플레이어 `+1B`
- 회수 기여자 `+0.15B`
- 실패한 도망자 `-0.5B`, 공동 보상 없음

### 단독 탈출

- 소지자 현상금 획득
- 남은 세 명은 20초 2-of-3 비상 발진
- 성공 시 각각 `+0.5B`

### 시간 초과

- 전원 `-0.25B`
- 현재 도망자 총 `-0.75B`

## 14. 쇼츠 사건 태그

- `TURN`: 최초 방향 전환
- `MAGNET_STEAL`: 계약자의 공개 강탈
- `STEAL`: 배터리 소유권 변경
- `BETRAY_AGAIN`: 회수자의 재배신
- `DOOR_HIT`: 격벽 차단
- `SLIDE_CHAIN`: 냉각수로 둘 이상 연쇄 미끄러짐
- `BUTTON_SAVE`: 버튼 2초 이내 저지
- `JAM_SAVE`: 추가된 재밍 시간 안에 저지
- `RICOCHET`: 배터리가 두 표면 이상 튕긴 뒤 소유권 변경
- `EMERGENCY`: 3인 비상 발진

클립 후보는 사건 전 8초·후 12초의 게임 로그와 함께 표시한다. 태그 발생 자체와 사람이 실제 공유할 장면인지는 분리 판정한다.

## 15. DDD 리듬

```text
2분대 협동 수리와 물리 사고
→ 지분·다음 장비 진행
→ 뜨거운 배터리 또는 손실 있는 계약
→ 역할 잠금과 30초 카운트다운
→ 운반자의 방향 전환
→ 관제자의 원격 사고 개입
→ 돌진·드롭·재배신
→ 마지막 버튼 니어미스
→ 공동 또는 단독·비상 발진
→ 20초 안의 정산과 즉시 재시작
```

## 16. 단계별 검증 모집단

### A단계 — 핵심 물리 검증

- 개발자 포함 4인 세션 12회
- 목적: 이동 시간, 도주 성공률, 관제 지배전략, 배터리 손바뀜
- A단계 수치 통과 전 그래픽·몬스터·맵 추가 금지

### B단계 — 블라인드 친구 그룹

- 개발자 없이 플레이하는 서로 다른 그룹 최소 20개, 총 80명
- 60초 설명 영상만 제공
- 목적: 첫 판 이해, 방치 역할, 실제 한 세션 더, 환불 의향
- 음성 녹화 동의 그룹만 사건 밀도 분석

### C단계 — 공개 데모

- 최소 100개의 완료된 4인 로비, 고유 호스트 60명 이상
- 한 그룹의 반복 세션은 최대 3개까지만 첫 사용자 지표에 포함
- 목적: 설치부터 방 생성까지 이탈, 첫 세션 완료, 실제 재실행과 기술 실패
- 친구 초대 실패와 매칭 실패는 재미 실패와 별도 기록

### D단계 — 쇼츠 시장 검증

- 서로 다른 사건 태그를 사용한 쇼츠 최소 20개
- 동일 편집 템플릿으로 게임 장면만 바꿔 비교
- UTM으로 상점 방문·위시리스트를 영상별 기록
- 최근 게임 쇼츠 20개의 중앙값을 기준선으로 사용
- 기존 게임 쇼츠 기준선이 없으면 첫 10개를 탐색 배치로 사용하고 다음 20개로 판정
- 조회·완주·UTM 상점 행동이 함께 상승한 태그만 출시 마케팅에 사용

## 17. 성공 기준

### A단계

- 첫 방향 선택 `6분 이내` 90%
- 첫 공개 도주·강탈 `8분 이내` 70%
- 30분당 배터리 손바뀜 `3회 이상`
- 전원 호위·전원 관제 중 한 전략의 비율 각각 70% 미만
- 자기장 강탈 성공률 30~60%

### B단계

- 네 명 모두 30분 안에 스포트라이트 경험 100%
- 규칙 설명 60초 뒤 핵심 이해 80% 이상
- 첫 세션 직후 실제 재시작 버튼 선택 70% 이상
- 관제 역할 중 `기다렸다` 응답 25% 미만
- 익명 60분 설문 환불 의향 15% 미만
- `볼 것을 다 봤다` 응답 25% 미만
- 7일 안에 같은 그룹 실제 재실행 30% 이상

### C단계

- 첫 로비 시작 성공률 90% 이상
- 첫 세션 완료율 75% 이상
- 완료 그룹의 즉시 재시작 50% 이상
- 치명 네트워크 오류 세션 5% 미만

### D단계

- 한 시간 원본 플레이당 편집 후보 5개 이상
- 사람이 실제 공유 판정한 클립 3개 이상/시간
- 한 시간당 서로 다른 태그 3종 이상
- 20개 중 최소 3개 쇼츠가 채널 최근 20개 중앙값의 조회수 3배 이상
- 조회만 오르고 UTM 상점 행동이 늘지 않은 태그는 실패 처리

### 100만 장 목표 해석

D단계는 `어떤 장면이 상대적으로 강한가`만 검증하며 100만 장 판매 가능성을 증명하지 않는다.

외부 영상 시청자의 가상 구매 전환율을 각각 0.1%, 0.25%, 0.5%로 놓은 단순 시나리오에서는 100만 구매에 각각 10억, 4억, 2억 회의 유효 조회가 필요하다. 이는 실제 전환율 주장이 아니라 목표 규모를 확인하기 위한 계산이다.

Steam 내부 추천과 친구 전파가 외부 조회 부담을 줄일 수 있지만, 공개 데모 UTM에서 실제 상점 방문·위시리스트·구매 전환을 얻기 전에는 100만 장 전망치를 만들지 않는다. 쇼츠 3배 기준은 개발 지속 여부를 판단하는 초기 신호로만 사용한다.

## 18. 웃음·클립 측정 규칙

- 관찰자 한 명이 영상 검토 중 사건 마커 기록
- 같은 사건에서 이어진 반응은 10초 안이면 1회
- 단순 대화량이 아니라 큰 웃음, 비명, 욕설, 전원 동시 반응만 강한 반응으로 집계
- 개발자 본인이 유도한 반응은 별도 표시하고 성공 기준에서 제외
- 클립 공유 판정은 플레이하지 않은 3명 이상에게 소리 없이 먼저 보여 2명 이상이 끝까지 보거나 다시 재생할 때 인정

목표는 강한 반응 10회/60분과 실제 공유 클립 3개/60분이다.

## 19. 폐기 기준

다음 중 하나가 해당 단계 표본을 채운 뒤에도 지속되면 맵이나 콘텐츠 추가보다 핵심 구조 폐기를 검토한다.

- 첫 8분 도주·강탈 세션 50% 미만
- 30분당 배터리 손바뀜 1회 미만
- 관제 역할의 대기 응답 40% 이상
- 전원 호위 또는 전원 관제 유지 70% 이상
- 운반 선택이 공동 또는 단독 한쪽으로 80% 이상 고정
- 같은 쇼츠 태그가 공유 판정 클립의 70% 이상
- 단독 탈출을 트롤로 평가 40% 이상
- B단계 즉시 재시작 50% 미만
- B단계 익명 환불 의향 25% 이상
- C단계 첫 세션 완료 60% 미만
- D단계에서 조회 상승 클립이 하나도 없음

## 20. 의도적으로 포기하는 범위

- 20시간 이상 장기 메타
- 랭크 시즌
- 출시 전 다수 맵
- 영구 능력치 성장
- 수십 개 역할
- 매주 신규 콘텐츠

첫 세션의 사건 밀도, 네 명의 능동적 참여, 초대·재시작 편의와 쇼츠 전환을 우선한다.
