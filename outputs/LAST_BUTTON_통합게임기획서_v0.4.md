# LAST BUTTON — 통합 게임 기획서 v0.4

작성일: 2026-07-12  
문서 상태: 현재 기획 기준안  
대상 플랫폼: Windows / Steam  
개발 전제: 2인 팀 + AI 보조  
플레이 인원: 4–6인 온라인 멀티플레이  
플레이 시간: 스테이지당 5–7분 / 3스테이지 세션 약 20–25분

## 1. 한 줄 소개

모두가 협력자로 시작하지만, 스테이지 중 누군가에게 공동 보너스를 훔칠 수 있는 비밀 계약이 도착한다. 계약을 수락한 플레이어는 정상 업무로 위장해 혼자 탈출을 준비하고, 동료들은 불완전한 단서와 대화로 그 선택을 밝혀내야 한다.

> 아무도 배신자가 아니었다. 누군가 계약을 받아들이기 전까지는.

## 2. 게임의 정체성

LAST BUTTON은 처음부터 정해진 임포스터를 찾는 게임이 아니다. 평범한 협력자가 커져가는 보상 때문에 배신자가 되는 순간과, 그 선택을 숨기기 위해 시작되는 거짓말을 다루는 사회적 도박 게임이다.

- 모든 플레이어가 같은 목표로 시작함
- 비밀 계약은 플레이 도중 자발적으로 수락함
- 아무도 계약하지 않은 스테이지도 존재함
- 초기 버전에서는 스테이지당 계약자 최대 한 명임
- 배신 준비는 정상 업무로 위장할 수 있어야 함
- 단서는 범인을 확정하지 않고 대화를 요구함
- 충성 플레이어에게 조사와 제재 수단이 존재함
- 배신 성공은 큰 세션 내 이득을, 실패는 실제 손실을 남김
- 살인과 탈락보다 속임수, 자원 절도, 추격과 몸싸움이 중심임

다른 소셜 디덕션 게임의 질문이 “누가 처음부터 적이었는가?”라면 LAST BUTTON의 질문은 다음과 같음.

> “우리와 함께 수리하던 사람이 언제 마음을 바꿨는가?”

## 4. 디자인 원칙

### 4.1 협동만으로도 재미있어야 함

배신자가 없는 스테이지에도 수리, 운반, 재난 대응, 구조와 물리적 사고가 재미를 만들어야 한다. 비밀 계약은 기본 협동을 대체하지 않고 긴장을 추가한다.

### 4.2 배신은 공식적으로 허용된 도박임

게임 시작 안내와 회사 방송은 블랙마켓 계약의 존재를 모든 플레이어에게 알린다. 계약을 수락한 플레이어는 규칙을 어긴 트롤이 아니라 큰 손실을 걸고 공식 선택지를 사용한 계약자다.

### 4.3 거짓말은 성공에 필요한 기술임

계약자는 버튼으로 바로 달릴 수 없다. 정상 업무처럼 보이는 위장 준비를 완료해야 하며, 다른 플레이어에게 자신의 이동과 작업 결과를 설명해야 한다.

### 4.4 단서는 대화를 만들고 시스템은 판결하지 않음

로그는 시간, 구역과 후보 행동만 제공한다. 누구의 말이 맞는지는 목격담, 동선과 제한된 보안 도구를 통해 판단한다.

### 4.5 추리에는 비용과 대응 결과가 있음

보안 검사는 강력하지만 횟수가 제한된다. 무고한 사람을 검사하면 공동 진행과 보안 자원을 잃기 때문에 무작정 모두를 검사할 수 없다.

### 4.6 배신당해도 플레이 전체가 무효가 되지 않음

개인 기여로 얻은 안전 임금은 보존한다. 계약자가 훔치는 대상은 추가 공동 보너스이며, 다른 플레이어는 일부 보험금과 비상 탈출 기회를 유지한다.

### 4.7 세션 이득은 이어지되 장기 전투력은 초기화함

스테이지에서 번 자산과 장비는 같은 3스테이지 세션에서만 유지한다. 세션이 끝나면 능력은 초기화하고 외형, 기록과 칭호만 장기 보상으로 남긴다.

## 5. 전체 세션 구조

```text
로비: 블랙마켓 계약 가능성 안내
  ↓
스테이지 1: 신뢰 형성 / 낮은 공동 금고
  ↓
휴게실: 정산 / 개인 장비 구매
  ↓
스테이지 2: 더 큰 금고 / 강화된 정보 혼란
  ↓
휴게실: 정산 / 장비 구매
  ↓
스테이지 3: 가장 큰 금고 / 최고 위험 계약
  ↓
세션 최종 순위와 리플레이
```

개인 탈출, 체포 또는 사망 여부와 관계없이 다음 스테이지에는 전원이 다시 참가한다. 세션 자산과 구매한 일회용 장비만 이어진다.

| 구분 | 스테이지 1 | 스테이지 2 | 스테이지 3 |
|---|---:|---:|---:|
| 제한 시간 | 6분 | 6분 | 7분 |
| 공동 금고 배수 | ×1.0 | ×1.5 | ×2.0 |
| 필수 시스템 | 2개 | 3개 | 3개 + 대형 재난 |
| 보안 검사 기본 충전 | 2회 | 2회 | 1회 |
| 정보 혼란 | 낮음 | 중간 | 높음 |
| 계약 유혹 | 낮음 | 중간 | 매우 높음 |

## 6. 한 스테이지의 흐름

### 0:00–1:30 — 공동 작업과 금고 축적

- 플레이어가 중앙 정비 구역에서 시작함
- 회사 방송이 필수 복구 목표와 공동 금고를 안내함
- 수리, 구조와 화물 회수로 안전 임금과 공동 금고가 증가함
- 이 구간에는 계약이 열리지 않아 기본 협력과 신뢰를 형성함

### 1:30–2:30 사이 — 유혹 발생

- 시간, 복구율과 금고 규모를 조합한 변동 시점에 비밀 제안이 도착함
- 모든 플레이어가 개인 화면에서 같은 계약 조건을 확인함
- 처음 수락한 한 명만 계약자가 됨
- 아무도 수락하지 않으면 계약자 없이 진행함
- 시스템은 `불법 통신 신호 감지`만 방송하며 실제 수락 여부는 공개하지 않음

### 2:00–4:30 — 위장, 단서와 의심

- 계약자는 정상 업무로 위장한 준비 행동 2개를 완료함
- 충성 플레이어는 수리, 재난 대응, 조사와 감시 사이에서 인력을 배분함
- 위장 행동과 재난이 비슷한 오류 흔적을 남겨 완전한 확신을 막음
- 플레이어는 근접 음성과 무전으로 동선과 결과를 설명함
- 충분한 의심이 생기면 제한된 보안 검사를 사용할 수 있음

### 마지막 60–90초 — 실행과 결말

- 위장 준비를 끝낸 계약자에게 간부용 키카드가 개방됨
- 키카드를 탈취하면 물리적 증거가 되어 추격이 시작될 수 있음
- 개인 포드를 충전하면 함선 전체에 명확한 경보가 발생함
- 마지막 버튼을 5초간 유지하면 개인 탈출에 성공함
- 다른 플레이어는 빗나갈 수 있는 돌진 밀치기, 키카드 압수와 전원 차단으로 저지함
- 공동 복구가 먼저 끝나면 중앙 탈출 구역을 유지해 공동 탈출함

### 결과 및 휴게실 — 약 30초

- 결정적 장면 8–12초를 리플레이함
- 안전 임금, 공동 금고 분배, 계약 보증금과 현상금을 정산함
- 플레이어가 다음 스테이지용 일회성 장비를 구매함
- 다음 스테이지의 환경 조건 일부를 예고함

## 7. 비밀 계약

### 7.1 제안 예시

> 블랙마켓 긴급 계약  
> 공동 보너스 금고를 탈취하면 금고의 70%와 특수 장비 선택권을 획득합니다. 계약 실패 시 현재 세션 자산의 25%를 잃습니다.

계약 조건과 수치는 모든 플레이어에게 동일하게 표시된다. 수락 여부만 개인 정보다.

### 7.2 계약 보증금

- 수락 시 현재 세션 자산의 25%를 보증금으로 잠금
- 스테이지 1처럼 자산이 부족하면 회사가 최소 보증금을 선지급하고 실패 시 부채로 처리함
- 성공 시 보증금 반환
- 체포 또는 시간 초과 실패 시 보증금 몰수
- 몰수액 일부는 검거 현상금으로 지급

### 7.3 계약자 수

- 초기 버전은 스테이지당 최대 한 명
- 첫 수락 이후 다른 플레이어의 제안은 조용히 만료됨
- 수락자가 있다는 사실은 공개하지 않음
- 6인 플레이에서 한 명이 지나치게 불리할 때만 복수 계약자를 후속 테스트함

## 8. 위장 준비

계약 성공에는 서로 다른 구역의 위장 행동 2개가 필요하다. 동일한 외형과 애니메이션을 가진 정상 작업과 비밀 작업을 한 콘솔 안에 배치한다.

### 8.1 전력 보정 또는 비밀 전용

- 정상 선택: 공동 복구율과 금고를 올림
- 비밀 선택: 개인 포드용 전력을 저장함
- 외부에서는 같은 레버와 수리 애니메이션으로 보임
- 비밀 선택은 복구율을 올리지 않고 전력 오차 로그를 남김

### 8.2 화물 검사 또는 인증 복사

- 정상 선택: 화물 보너스를 공동 금고에 추가함
- 비밀 선택: 간부용 발사 인증을 복사함
- 외부에서는 같은 스캐너 작업으로 보임
- 비밀 선택은 해당 시간대의 출입자 후보만 로그에 남김

### 8.3 키카드 개방

- 두 위장 행동을 완료하면 계약자만 간부용 키카드를 꺼낼 수 있음
- 키카드를 꺼낸 이후에는 누구나 물리적으로 확인하고 빼앗을 수 있음
- 키카드 권한은 소유자가 아니라 계약 상태에 연결됨
- 충성 플레이어가 키카드를 확보하면 보안함에 반환하거나 검사 증거로 사용할 수 있음
- 다른 비계약자가 키카드를 주워 즉시 새 배신자가 되는 규칙은 폐기함

## 9. 정보와 단서

### 9.1 보안 로그

로그는 다음 정보만 제공함.

- 비정상 작업이 발생한 구역
- 약 15초 단위의 시간 범위
- 해당 시간대에 단말기를 사용한 후보 인원 수
- 오류가 사람 행동인지 재난인지에 대한 불확실한 분류

플레이어 이름 하나를 확정적으로 표시하지 않는다.

### 9.2 CCTV

- 일부 주요 통로만 촬영함
- 정전, 증기와 몬스터로 화면이 가려질 수 있음
- 실시간 감시에는 한 명의 작업 인력이 필요함
- 스테이지 결과에서만 결정적 장면을 선명하게 다시 보여줌

### 9.3 환경 사건의 정보 역할

환경 사건은 단순 방해가 아니라 증거 신뢰도와 동선을 바꿔야 한다.

- 정전: CCTV와 출입 로그 일부 중단
- 산소 누출: 플레이어를 여러 구역으로 강제 분산
- 문 고장: 알리바이 동선을 변경
- 전력 불안정: 비밀 전용과 유사한 오류 발생
- 몬스터: 자원 분실과 설비 파손으로 거짓 배신 흔적 생성

## 10. 보안 검사와 제재

### 10.1 2인 보안 검사

- 두 명이 보안 스캐너에서 같은 용의자에게 동의해야 시작함
- 스테이지당 기본 1–2회만 사용할 수 있음
- 검사는 계약 수락 여부를 확정함
- 한 명이 단독으로 다른 플레이어를 장시간 구금할 수 없음

### 10.2 계약자 적발

- 계약 즉시 취소
- 계약 보증금 몰수
- 키카드와 저장한 포드 전력 초기화
- 검거에 참여한 플레이어에게 현상금 지급
- 적발된 플레이어는 탈락하지 않고 수리로 복귀해 안전 임금을 계속 획득함
- 같은 스테이지에서는 다시 계약할 수 없음

### 10.3 무고한 플레이어 검사

- 공동 복구율 8% 감소
- 보안 검사 충전 1회 소모
- 검사 주도자의 기여 등급 일부 감소
- 무고한 플레이어는 조작 불능이나 장기 감금 상태가 되지 않음

### 10.4 실행 단계 대응

- 키카드 압수
- 계약자 위치 추적기 부착
- 포드실 출입 권한 20초 제한
- 포드 전력 차단
- 돌진 밀치기로 버튼 중단

수치는 플레이테스트로 조정하되, 제재는 계약을 위험하게 만들면서도 플레이어를 게임에서 제외하지 않아야 한다.

## 11. 보상 경제

### 11.1 안전 임금

다음 행동으로 개인별 안전 임금을 얻으며 배신 성공 여부와 관계없이 보존한다.

- 수리 기여
- 화물 운반
- 동료 구조
- 재난 대응
- 키카드 회수
- 올바른 계약자 검거

### 11.2 공동 보너스 금고

- 협동 작업의 속도와 품질에 따라 커짐
- 공동 탈출 시 생존자에게 균등 분배함
- 개인 기여는 안전 임금에서 반영하고 금고는 지나친 경쟁을 막기 위해 균등 분배함

### 11.3 계약 성공

- 계약자가 공동 금고의 70%를 획득함
- 남은 30%는 다른 플레이어에게 보험금으로 분배함
- 계약 보증금을 돌려받음
- 다음 스테이지에서 블랙마켓 장비 선택 후보가 증가함
- 남은 팀에게 60초의 비상 복구 시간이 주어짐

### 11.4 계약 실패

- 계약 보증금 25%를 잃음
- 공동 금고는 보존됨
- 몰수액 일부가 검거 현상금이 됨
- 안전 임금은 유지해 실패 후 고의적인 게임 방해를 줄임

### 11.5 세션 내 장비

- 추가 질주 배터리
- 휴대용 문 잠금 장치
- 1회용 보안 스캐너
- 키카드 방향 탐지기
- 몬스터 유인기
- 가짜 경보 장치
- 수리 과충전 도구
- 휴대용 CCTV

장비는 강한 영구 능력보다 새로운 선택과 거짓말을 만드는 일회용 도구로 설계한다. 3스테이지 세션이 끝나면 모두 초기화한다.

### 11.6 장기 보상

- 우주복과 헬멧
- 바이저 색과 음성 반응 효과
- 포드 외형
- 버튼 작동 애니메이션
- 승리 포즈, 칭호와 통계 배지

장기 이동 속도, 수리 속도와 밀치기 힘 강화는 사용하지 않는다.

## 12. 승패와 결과

### 공동 성공

- 필수 시스템을 복구하고 공동 탈출 구역을 유지함
- 안전 임금과 공동 금고 지분을 획득함
- 계약자가 없거나 계약을 포기한 스테이지도 정상적인 성공으로 인정함

### 계약자 체포 후 공동 성공

- 계약자를 정확히 검사하고 공동 탈출함
- 공동 금고와 검거 현상금을 획득함
- 적발된 계약자도 이후 수리에 기여한 안전 임금은 유지함

### 개인 탈출 성공

- 위장 행동 2개, 키카드, 포드 충전과 마지막 버튼을 완료함
- 공동 금고 70%와 계약 보너스를 획득함
- 다른 플레이어는 보험금과 60초 비상 복구 기회를 얻음

### 전체 실패

- 제한 시간 안에 공동 또는 개인 탈출이 이루어지지 않음
- 안전 임금 일부만 확정하고 공동 금고는 소멸함

## 13. 근접 음성과 거짓말

근접 음성은 부가 기능이 아니라 핵심 게임 시스템이다.

- 거리에 따라 음량 감소
- 벽과 닫힌 문이 음성을 약화
- 무전기는 멀리 있는 플레이어와 제한적으로 연결
- 정전과 통신 고장이 알리바이 공유를 방해
- 계약자는 작업 결과와 이동 이유를 직접 설명해야 함
- 사망자와 개인 탈출자는 생존자에게 정보를 전달할 수 없음

초기 네트워크 테스트는 Discord로 가능하지만, 위장과 알리바이 검증 단계부터 근접 음성을 우선 구현한다.

## 14. 물리적 대치

- 이동, 질주, 물체 운반과 던지기
- 쓰러진 동료 구조
- 조준 방향 단거리 돌진 밀치기
- 돌진은 빗나가거나 벽에 충돌할 수 있음
- 키카드 소지자가 밀리면 키카드를 떨어뜨림
- 마지막 버튼은 5초 유지가 필요함
- 반복 밀치기는 후딜레이와 재사용 대기시간으로 제한함
- 직접적인 살인 무기와 영구 탈락은 초기 범위에서 제외함

물리 대치는 추리를 대체하지 않고, 거짓말이 끝나고 정체가 드러난 뒤 발생하는 결말이어야 한다.

## 15. 의미 있는 판별 변화

맵 변화는 위치만 바꾸지 않고 판단 규칙을 바꿔야 한다.

### 유효한 변화

- CCTV가 작동하지 않는 스테이지
- 출입 로그가 정전 중 기록되지 않는 스테이지
- 포드실 지름길이 열리지만 몬스터 둥지가 가까운 배치
- 수리와 위장 콘솔의 후보 구역이 달라지는 배치
- 보안 검사 충전이 1회뿐인 고위험 규칙
- 몬스터 파손과 비밀 전용이 같은 오류를 남기는 규칙

### 유효하지 않은 변화

- 상자 위치만 변경
- 벽 색만 변경
- 같은 판단을 더 긴 이동으로 반복
- 플레이어 선택과 정보에 영향을 주지 않는 장식 랜덤화

## 16. 몬스터의 위치

몬스터는 초기 핵심 검증 이후 추가한다. 몬스터의 목적은 전투가 아니라 다음을 만드는 것임.

- 플레이어 강제 분산
- 거짓 배신 흔적
- 자원 분실
- 구조를 위한 위험한 협동
- 계약자가 이용할 수 있는 혼란

몬스터는 회피만 하는 장애물이 아니라 유인, 격리, 일시 제압과 역이용이 가능해야 한다. 몬스터가 비밀 계약과 마지막 버튼보다 결말을 지배하면 안 된다.

## 17. 도파민 포인트

한 스테이지의 감정 최고점을 다음처럼 분산한다.

1. 협동 수리 완벽 성공
2. 희귀 화물 발견과 금고 상승
3. 비밀 계약 도착
4. 계약 수락 여부의 짧은 고민
5. 위장 행동을 들키지 않고 완료
6. 의심스러운 보안 로그 발견
7. 거짓말로 검사를 피하거나 알리바이를 설득
8. 무고한 플레이어를 잘못 검사한 사고
9. 실제 계약자 검사 성공
10. 포드 충전 경보와 추격 시작
11. 돌진 명중으로 키카드 드롭
12. 마지막 버튼 직전 저지 또는 극적 탈출
13. 보상 정산과 현상금 공개
14. 다음 스테이지 장비 선택

## 18. 플레이테스트 지표

초기 목표 범위이며 재미 테스트 후 조정한다.

| 지표 | 목표 |
|---|---:|
| 계약 제안 대비 수락률 | 30–60% |
| 계약 성공률 | 수락 건의 20–40% |
| 계약 실행 전 의심 대화 발생 | 수락 판의 70% 이상 |
| 무고한 보안 검사 | 전체 검사의 20–40% |
| 배신 실행 시 키카드 소유권 변경 | 평균 1회 이상 |
| 돌진 명중률 | 35–55% |
| 판당 강한 웃음·비명·언쟁 | 2회 이상 |
| 3판 후 한 판 더 의향 | 70% 이상 |

### 테스트 질문

1. 계약을 수락하거나 거절한 이유는 무엇인가?
2. 언제 누구를 의심했고 어떤 단서가 영향을 주었는가?
3. 거짓말하거나 알리바이를 설명할 필요를 느꼈는가?
4. 보안 검사를 사용할 때 실제로 고민했는가?
5. 배신이 전략으로 느껴졌는가, 트롤링으로 느껴졌는가?
6. 안전 임금과 위험 보상의 차이를 이해했는가?
7. 다음 스테이지 장비 때문에 결과가 중요하게 느껴졌는가?
8. 다음 판에 다른 선택을 시도하고 싶은가?

## 19. 위험과 대응

### 모두가 계약을 수락하려 함

- 최초 한 명만 계약 가능
- 보증금과 실패 손실 강화
- 공동 성공의 안정적인 기대값 유지

### 아무도 계약하지 않음

- 공동 금고가 커질수록 계약 보너스 증가
- 스테이지 2와 3에서 특수 장비 선택권 강화
- 강제 임포스터는 도입하지 않음

### 보안 검사가 사실상 투표 추방이 됨

- 2인 합의와 제한 횟수 요구
- 무고한 검사에 팀 손실 부여
- 적발된 플레이어를 탈락시키지 않음

### 계약자가 트롤로 인식됨

- 로비와 튜토리얼에서 계약이 공식 승리 경로임을 명시
- 다른 플레이어의 안전 임금 보존
- 공동 금고 30% 보험금 지급
- 결과 화면에서 전략적 성공과 실패를 명확히 연출

### 성공한 계약자가 계속 강해짐

- 보상은 3스테이지 세션에서만 유지
- 장비는 일회용과 선택 확장 중심
- 영구 능력 강화 금지

### 협동이 배신 대기 노동이 됨

- 수리 자체에 짧은 성공, 사고와 희귀 보상 배치
- 계약 개방 전 신뢰 형성 구간 유지
- 개인 안전 임금으로 꾸준한 성취 제공

## 20. 제작 우선순위

### 기획 검증 프로토타입

- 비밀 계약 제안과 최초 수락
- 정상 작업과 외형이 같은 위장 작업 2종
- 불완전한 보안 로그
- 2인 보안 검사와 오판 비용
- 안전 임금과 공동 금고
- 계약 성공, 실패와 보험금
- 다음 스테이지 장비 구매
- 기존 키카드, 포드, 돌진과 마지막 버튼 연결

### 그래픽 버티컬 슬라이스

- 포드실과 주변 복도 목표 품질 완성
- 캐릭터와 기본 애니메이션
- 위장 작업이 외부에서 동일하게 보이는 연출
- 정전, 로그, 검사, 포드 경보와 충돌 피드백
- 금속 발소리, 함선 배경음과 근접 음성

### 4인 온라인 테스트

- 이동, 상호작용과 물리 동기화
- 계약 상태의 서버 권한 및 비공개 전송
- 보안 로그와 검사 판정 동기화
- 키카드, 포드와 돌진 서버 판정
- 3스테이지 정산과 재대결

## 21. 확정 사항

- 4–6인 온라인 멀티플레이
- 스테이지당 5–7분, 세션당 3스테이지
- 모든 플레이어가 같은 편으로 시작함
- 고정 임포스터 없음
- 회사가 모든 플레이어에게 비밀 계약을 제안함
- 최초 수락자 한 명만 초기 계약자가 됨
- 아무도 수락하지 않은 스테이지 가능
- 계약 수락 여부는 비공개
- 계약 성공에는 위장 행동 2개가 필요함
- 단서는 불완전하며 보안 검사는 2인 합의가 필요함
- 적발된 계약자는 탈락하지 않음
- 안전 임금은 배신 결과와 무관하게 보존됨
- 계약 성공자는 공동 금고 70%를 획득함
- 나머지 30%는 보험금으로 분배함
- 계약 실패 시 세션 자산 25% 보증금 몰수
- 세션 자산은 다음 스테이지 장비에 사용함
- 세션 종료 시 능력과 장비 초기화
- 키카드는 실행 단계의 물리 증거임
- 마지막 버튼은 5초 유지가 필요함
- 돌진 밀치기는 빗나갈 수 있음
- 근접 음성은 핵심 기능임
- 몬스터는 핵심 계약 루프 검증 이후 추가함

## 22. 플레이테스트 후 확정할 사항

- 계약 제안 정확한 시점과 표시 방식
- 계약 수락 제한 시간
- 계약 보증금 25%의 적절성
- 금고 독식 70%와 보험금 30% 비율
- 위장 작업의 수와 소요 시간
- 로그가 표시할 후보 정보의 정확도
- 보안 검사 충전 횟수
- 무고한 검사 복구율 손실 8%
- 적발 후 출입 제한 도입 여부
- 일회용 장비 목록과 가격
- 개인 탈출 후 비상 복구 시간
- 스테이지별 계약 수락률과 성공률 목표
- 6인 플레이에서 복수 계약자 허용 여부

## 23. 현재 프로토타입과의 차이

현재 Unity 프로토타입은 v0.2 규칙을 구현한다.

- 복구율 40%에서 키카드를 직접 탈취함
- 키카드 탈취가 사실상 배신 시작임
- 위장 행동과 비밀 계약 UI가 없음
- 불완전한 보안 로그와 2인 검사가 없음
- 보상 소비처와 3스테이지 연결이 없음

프로토타입 코드는 즉시 변경하지 않는다. v0.4 기획 검토와 그래픽 방향 확정 후 `기획 검증 프로토타입` 항목부터 순서대로 반영한다.

## 24. 최초 검증 장면

첫 제작 목표는 다음 장면이 사람 플레이에서 자연스럽게 발생하는지 확인하는 것임.

> 불법 통신 신호가 감지되지만 실제 계약자가 있는지는 아무도 모른다. 전력실에서 함께 작업한 두 사람 중 한 명은 수리를 했다고 주장하지만 복구율은 오르지 않았다. 팀은 마지막 보안 검사 한 번을 사용할지 논쟁한다. 무고한 사람을 의심하는 사이 진짜 계약자가 화물실에서 인증을 복사하고 키카드를 꺼낸다. 포드 경보가 울리고 모두가 달려오지만, 버튼 앞 첫 돌진이 벽에 빗나간다. 두 번째 플레이어가 마지막 순간 키카드를 떨어뜨려 계약을 저지한다.

이 장면에서 계약자가 거짓말할 이유, 충성 플레이어가 의심하고 확인할 이유, 들킨 뒤 물리적으로 저지할 이유와 결과를 다음 스테이지까지 기억할 이유가 모두 발생해야 한다.
