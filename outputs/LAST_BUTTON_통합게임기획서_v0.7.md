# LAST BUTTON — 통합 게임 기획서 v0.7

작성일: 2026-07-12  
문서 상태: 반복 검증안  
검증 대상: 마이크를 사용하는 친구 4명  
플레이 시간: 5–7분 × 3스테이지

## 1. 핵심 정의

네 명의 계약직 정비공이 고장 난 함선을 함께 살리고 각자의 고용 유지 할당량을 채운다. 스테이지 도중 누구든 블랙 계약을 제안받을 수 있으며, 계약자는 전원이 참여하는 위상 체크섬에서 거짓 기호를 보고해 개인 포드 전력을 빼돌린다.

최종 목표는 다른 플레이어를 이기는 단독 순위가 아니라 3스테이지 종료 전 자신의 고용 할당량을 달성하는 것임. 여러 명이 함께 할당량을 달성할 수 있고 다른 플레이어의 지분을 낮출 직접적인 이득이 없음.

## 2. v0.6 치명 결함 제거

| v0.6 결함 | v0.7 규칙 |
|---|---|
| 선두가 팀 실패로 순위를 고정할 수 있음 | 단독 1위 폐기, 개인 할당량 달성으로 전환 |
| 특정 판독자만 계약을 실행할 수 있음 | 전원 참여 체크섬에서 누구나 자신의 기호를 거짓 보고 가능 |
| 역할 사전 고정으로 계약 봉쇄 | 체크섬에는 네 명 모두 참여하고 고정 역할이 없음 |
| 계약자가 빠지면 발진 역할 부족 | 세 설비 중 두 설비만 유지하면 발진 진행 |
| 플레이어 제재가 경쟁 방해로 악용 | 플레이어 제재 없음, 오염된 회로 하나만 재보정 가능 |
| 두 로그 교집합으로 계약자 확정 | 단일 체크섬과 두 개의 2인 회로만 사용 |

## 3. 고용 할당량

### 개인 목표

- 세션 시작 시 모든 플레이어의 할당량은 `3B`
- 공동 성공 한 번은 각 플레이어에게 `1B`
- 합법 개인 업무는 스테이지당 최대 `0.25B`
- 개인 작업 성과는 스테이지당 최대 `0.15B`
- 완전한 공동 성공 3회면 누구나 할당량을 달성함
- 한 스테이지를 실패해도 합법 업무와 블랙 계약으로 부족분을 만회할 수 있음
- 할당량을 초과한 지분은 세션 장비와 장기 외형 재화로만 사용함

여러 플레이어가 동시에 고용 유지에 성공할 수 있음. 다른 사람의 할당량 달성을 방해해도 자신의 지분은 증가하지 않음.

### 팀 실패

- 함선 전체 실패 시 해당 스테이지 공동 보너스 `1B`를 아무도 받지 못함
- 이미 획득한 합법 업무와 개인 성과는 절반만 지급함
- 고의적인 팀 실패는 자신의 할당량 달성도 어렵게 만듦

## 4. 합법 개인 업무

합법 업무는 공동 목표와 같은 방향의 비독점 조건만 사용함.

- 서로 다른 두 협동 수리에 참여
- 화물을 손상 없이 공동 운반
- 동료 구조
- 두 설비에서 정상 또는 완벽 수리 달성
- 최종 발진 중 두 번의 설비 교대 성공

마지막 입력 독점, 희귀 자원 개인 보관, 수리 지연과 타인의 실패를 요구하는 업무는 사용하지 않음.

플레이어는 자신의 합법 업무를 공개할 수 있지만 블랙 계약 보유 여부는 증명되지 않음. 공개 시 합법 업무 보너스 25%를 잃음.

## 5. 블랙 계약

### 제안

- 순위와 무관하게 네 명 모두 제안 대상임
- 무작위 한 명에게 12초간 비공개 제안함
- 거절 시 다른 무작위 플레이어에게 한 번 더 제안함
- 두 번 모두 거절하면 해당 스테이지는 계약자 없이 진행함
- 이전 계약자는 다음 스테이지에도 대상이 될 수 있음
- 수락 15–25초 후 계약자 존재 경보만 발생함

### 보상

- 성공: `2.5B`
- 실패 또는 공동 발진 선행 성공: `-0.5B`
- 계약 성공 시 다른 플레이어는 각각 `0.75B`
- 계약 여부와 관계없이 개인 성과 지분은 보존됨

공동 성공의 기대값은 `1B`임. 계약은 평균적으로 유리한 선택이 아니라 현재 할당량 부족을 한 번에 만회하는 고분산 선택임.

## 6. 전원 위상 체크섬

공동 최종 발진 전에 네 명 모두가 참여해야 하는 필수 작업임.

### 기본 절차

1. 플레이어마다 개인 패널에 비공개 기호 하나가 표시됨
2. 두 명씩 `청색 회로`와 `적색 회로`에 자동 배정됨
3. 각 플레이어는 자신의 기호를 음성 또는 핑으로 보고함
4. 각자 보고한 기호를 자신의 패널에 직접 입력함
5. 네 입력이 합쳐져 위상 체크섬을 구성함

### 정상 플레이

비계약자는 표시된 정상 기호만 입력할 수 있음. 잘못 입력하면 즉시 일반 오류가 발생하고 다시 입력해야 함.

### 계약 플레이

계약자의 패널에는 정상 기호와 블랙 계약용 대체 기호가 함께 표시됨.

- 정상 기호를 입력하면 계약을 포기하고 공동 진행함
- 대체 기호를 입력하려면 다른 플레이어에게 대체 기호를 정상 기호인 것처럼 보고해야 함
- 대체 기호는 시스템상 정상 입력으로 처리되어 즉시 오류가 발생하지 않음
- 블랙 계약 준비는 이 한 번의 거짓 보고로 완료됨
- 계약자는 어느 회로에 배정되어도 실행할 수 있음

전원 참여 작업이므로 계약자만 특정 역할을 요청하거나 다른 사람이 협조를 거부하는 방식으로 봉쇄할 수 없음.

## 7. 지연된 회로 이상

대체 기호 입력 20–35초 후 다음 현상이 발생함.

- 공동 전력 감소
- 개인 포드 전력 활성화
- 불법 전력 이상 방송
- 청색 또는 적색 회로 중 하나가 오염됐다는 센서 경고

계약자가 속한 회로가 실제 오염 회로지만 센서 두 개가 상충하는 결과를 출력함.

예시:

```text
센서 A: 청색 회로 오염 가능성 65%
센서 B: 적색 회로 오염 가능성 60%
```

각 회로에는 두 명이 있으므로 한 회로를 의심해도 계약자 한 명으로 확정되지 않음. 모든 플레이어가 체크섬에 참여했기 때문에 두 작업 로그의 교집합도 존재하지 않음.

## 8. 충성 플레이어의 회로 대응

팀은 스테이지당 한 번만 `회로 재보정`을 실행할 수 있음.

- 청색 또는 적색 회로 중 하나를 선택함
- 두 명이 12초 동안 재보정 작업을 해야 함
- 실제 오염 회로를 선택하면 포드 전력을 제거하고 계약은 실패함
- 정상 회로를 선택하면 포드 전력은 유지되고 12초를 잃음
- 선택 결과는 포드 전력이 유지됐는지만 알려주며 계약자 이름은 공개하지 않음
- 계약자는 대화에서 반대 회로를 의심하도록 유도할 수 있음

플레이어 한 명을 검사, 감금하거나 출입 정지하는 기능은 사용하지 않음.

## 9. 키카드와 실행 단계

계약 준비가 완료되면 계약자에게만 키카드 보관함 위치가 표시됨.

- 키카드는 두 회로 구역과 떨어진 공용 장소에 생성됨
- 키카드를 꺼내면 물리 오브젝트로 모든 플레이어에게 보임
- 누구나 돌진으로 떨어뜨리고 회수할 수 있음
- 비계약자는 키카드를 5초간 보안함에 반환할 수 있음
- 계약자는 포드 전력을 유지한 채 마지막 버튼으로 이동함

회로 재보정이 성공하면 키카드가 있어도 포드를 사용할 수 없음.

## 10. 공동 최종 발진

최종 30초에는 세 설비가 존재함.

- 원자로 안정기
- 항법 계산기
- 화물 고정 장치

### 진행 규칙

- 세 개 중 두 개 작동: 발진 게이지 정상 증가
- 세 개 모두 작동: 발진 게이지 빠르게 증가
- 한 개 이하 작동: 발진 게이지 감소
- 설비는 한 명이 짧은 조작 후 6초 동안 자동 유지됨
- 한 플레이어가 여러 설비 사이를 이동하며 갱신할 수 있음

### 계약자가 없는 판

네 명이 수리, 돌발 고장과 추가 화물을 분담함.

### 계약자가 실행하는 판

- 충성 플레이어 두 명이 설비 두 개를 유지할 수 있음
- 남은 한 명은 회로 재보정, 키카드 회수 또는 추격을 수행함
- 필요하면 설비 담당 한 명이 짧게 이탈한 뒤 6초 안에 복귀함

계약자 한 명의 이탈만으로 공동 발진이 자동 실패하지 않음.

## 11. 스테이지 종료

### 공동 성공

- 공동 발진 게이지가 먼저 완료됨
- 모든 플레이어가 `1B`와 개인 성과를 획득함
- 계약자는 계약 실패 `-0.5B` 적용

### 블랙 계약 성공

- 계약자가 포드 전력과 키카드를 확보함
- 마지막 버튼을 5초간 유지함
- 버튼 성공 즉시 스테이지 종료
- 계약자는 `2.5B`와 개인 성과를 획득함
- 다른 플레이어는 각각 `0.75B`와 개인 성과를 획득함

### 전체 실패

- 제한 시간 안에 어느 발진도 성공하지 못함
- 공동 보너스 없음
- 개인 성과는 절반 지급
- 수락한 계약자는 `-0.5B`

스테이지가 종료되면 전원이 다음 스테이지에 즉시 참가하며 관전 공백은 없음.

## 12. 코미디 수리

### 냉각수 호스

- 한 명이 호스를 잡고 다른 한 명이 밸브를 잠금
- 놓치면 정해진 범위의 물줄기가 플레이어를 밀치고 바닥을 미끄럽게 함
- 실패해도 밸브 진행 일부 유지
- 정상과 완벽 수리 판정 존재

### 압력 밸브

- 한 명은 압력 숫자를 보고 다른 한 명은 밸브만 조작함
- 과압 시 1–3초 증기 충격과 물건 드롭
- 목표 구간 유지 시간으로 정상과 완벽 수리 판정

### 원칙

- 실행 10–20초
- 실패해도 전체 초기화 없음
- 물리 사고는 짧고 회복 가능함
- 개인 지분 손실보다 팀 진행 지연과 장면 생성 중심
- 연속 로프 물리보다 호스트가 판정하는 제한 상태와 충격 사용

초기 검증은 두 작업만 사용하며 반복 피로를 별도 측정함.

## 13. DDD 리듬

```text
협동 수리 성공과 짧은 사고
→ 개인 할당량 상승
→ 합법 업무 진행
→ 블랙 계약 유혹 또는 계약 경보
→ 전원 체크섬과 거짓 기호
→ 지연된 회로 이상
→ 둘 중 하나에 재보정 선택
→ 공동 발진과 키카드 추격
→ 마지막 버튼 니어미스
→ 실제 계약자 공개와 할당량 정산
```

계약자가 없는 판에는 수리 숙련, 합법 업무, 최종 발진과 할당량 달성이 핵심 보상임.

## 15. 적대 검증 지표

### 치명 메타 확인

- 계약 경보 후 특정 순위나 역할을 일괄 배제하는 행동 20% 미만
- 회로 정보만으로 결과 공개 전 계약자를 확정한 비율 75% 미만
- 계약자가 거짓 기호 또는 거짓 회로 주장을 사용한 판 90% 이상
- 회로 재보정이 항상 같은 정답 행동으로 선택되지 않음
- 팀 실패를 고의로 유도하는 것이 개인적으로 이득이라고 답한 플레이어 10% 미만

### 재미

- 계약자가 없는 판의 재플레이 의향 60% 이상
- 충성과 계약 역할 재미 차이 1점 이내
- 수리만 반복했다고 답한 플레이어 25% 미만
- 판당 기억나는 클러치 또는 사고 평균 2개 이상
- 3스테이지 후 재대결 의향 60% 이상

## 16. 폐기 기준

- 체크섬 구조가 몇 판 안에 계약자를 자동 확정함
- 계약 경보 후 체크섬 참여를 거부하는 전략이 반복됨
- 계약자가 거짓 정보를 전달하지 않고도 준비를 완료함
- 회로 재보정이 계약을 항상 무력화하거나 항상 무시됨
- 개인 할당량 때문에 정상 협동 방해가 반복됨
- 계약자가 없는 판에서 수리가 재미없음
- 계약 성공이 전략보다 트롤링으로 평가됨
- 3스테이지 후 재플레이 의향이 절반 미만임

## 17. 현재 프로토타입과의 차이

현재 Unity 프로토타입은 v0.2 규칙을 구현함. v0.7의 개인 할당량, 전원 체크섬, 회로 재보정, 코미디 수리와 2-of-3 발진은 구현하지 않음.

반복 기획 검증이 끝날 때까지 프로토타입 코드를 변경하지 않음.
