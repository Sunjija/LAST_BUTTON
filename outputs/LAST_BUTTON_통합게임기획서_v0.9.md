# LAST BUTTON — 통합 게임 기획서 v0.9

작성일: 2026-07-12  
문서 상태: 반복 검증안  
대상: 마이크를 사용하는 친구 4명  
플레이 시간: 5–7분 × 3스테이지

## 1. 핵심 구조

네 명이 함선을 수리하며 각자 `3.5B`의 고용 할당량을 채운다. 공동 성공은 모두에게 `1B`를 주는 안정 경로다. 누구든 블랙 계약을 제안받을 수 있고, 계약자는 전원 위상 체크섬에서 거짓 기호를 보고해 성공 `2.5B`, 실패 `-0.5B`의 역전 도박을 시도한다.

다른 사람을 이기는 단독 순위는 없으며 여러 명이 함께 고용을 유지할 수 있음. 타인의 성과를 낮추는 행위는 자신의 할당량을 높이지 않음.

## 2. 세션 경제

- 개인 할당량: `3.5B`
- 공동 성공: `1B`
- 합법 업무와 개인 성과: 스테이지당 합계 최대 `0.25B`
- 계약 성공: 계약자 `2.5B`, 나머지 각각 `0.75B`
- 계약 실패: 계약자 `-0.5B`, 공동 성공자는 `1B`
- 전체 실패: 공동 보너스 없음, 개인 성과 절반, 전원 `-0.25B`
- 세션 종료 전까지 할당량 달성은 확정되지 않음
- 할당량 초과분은 외형 재화로만 전환됨

정상적인 공동 성공 세 번과 평균 `0.5B` 이상의 개인 성과로 할당량을 달성함. 다른 사람의 계약이 한 번 성공하면 세 스테이지 개인 성과를 최대치에 가깝게 달성해야 할당량을 지킬 수 있음.

## 3. 비독점 충성 플레이

- 협동 수리는 참여자 모두에게 같은 기본 성과 지급
- 완벽 수리 보너스도 참여자 모두에게 지급
- 합법 업무는 공동 목표와 같은 방향의 조건만 사용
- 마지막 입력, 희귀 자원 독점과 고의 지연으로 성과를 얻을 수 없음
- 유형별 성과에는 상한이 있어 반복 파밍 불가
- 함선 실패는 자신에게도 `-0.25B`를 부과함
- 발진 경보 전 고의 밀치기는 성과 차감

합법 업무 예시는 협동 수리 두 종류 참여, 무손상 공동 운반, 동료 구조와 발진 설비 교대임.

## 4. 계약 제안

- 네 명 모두 제안 대상
- 무작위 한 명에게 12초 비공개 제안
- 거절 시 다른 무작위 한 명에게 마지막 제안
- 모두 거절하면 계약자 없는 스테이지
- 수락 15–25초 후 계약자 존재 경보
- 제안 대상, 거절 여부와 개인 할당량은 비공개

## 5. 전원 위상 체크섬

공동 발진 전에 네 명 모두 참여함.

1. 각 플레이어에게 정상 기호 하나가 비공개 표시됨
2. 각 플레이어는 자신에게 보인 기호를 음성 또는 핑으로 보고함
3. 보고한 기호를 자신의 패널에 입력함
4. 비계약자의 다른 기호 입력은 즉시 오류 처리됨
5. 계약자에게는 정상 기호와 계약 기호가 함께 표시됨
6. 계약 기호는 즉시 오류 없이 정상 성공처럼 처리됨
7. 계약자는 계약 기호를 정상 기호라고 거짓 보고해야 함

고정 역할이 없고 전원 참여이므로 역할 배정이나 협조 거부로 계약을 봉쇄할 수 없음.

## 6. 인과적 2인 후보 생성

계약 기호 입력 20–35초 후 비인가 전력 이상이 발생함.

시스템은 다음 두 패널을 `불일치 후보`로 표시함.

- 실제 계약 기호가 입력된 계약자 패널 1개
- 다른 회로에서 무작위로 선택된 무고한 패널 1개

회사는 노후 센서의 교차 오작동 때문에 두 패널 중 하나만 실제 오염이라고 안내함.

### 공개 정보

- 후보 두 명의 이름
- 두 후보가 당시 공개적으로 보고한 기호
- 두 후보의 입력 완료 순서가 `빠름`, `보통`, `늦음` 중 하나로 표시됨
- 두 후보 모두 자신의 패널 입력은 정상이라고 주장할 수 있음

### 비공개 정보

- 계약자는 자신이 계약 기호를 입력했다는 사실을 앎
- 무고한 후보는 자신이 정상 기호를 입력했다는 사실을 앎
- 다른 두 플레이어는 두 후보의 주장, 입력 당시 행동과 기존 동선을 바탕으로 판단함

무작위 확률 숫자와 두 로그 교집합은 사용하지 않음. 후보는 항상 정확히 두 명이며 한 명은 계약자, 한 명은 무고함.

## 7. 패널 재보정

- 팀은 후보 패널 둘 중 하나만 재보정할 수 있음
- 두 명이 12초 동안 작업함
- 계약자 패널 선택: 포드 전력 제거, 계약 실패
- 무고한 패널 선택: 포드 전력 유지, 12초 손실
- 계약자 신원은 결과 화면 전까지 시스템이 직접 선언하지 않음

계약자는 무고한 후보가 거짓말한다고 주장하고 자신의 기호와 입력 행동을 설명해야 함. 무고한 후보도 같은 조건에서 자신의 정상 입력을 설명함.

## 8. 키카드와 발진 경쟁

### 계약 실행

- 포드 전력이 유지되면 계약자에게만 키카드 보관 위치 표시
- 키카드를 꺼내면 모두에게 보이는 물리 오브젝트가 됨
- 돌진으로 키카드를 떨어뜨릴 수 있음
- 비계약자는 5초간 키카드를 보안함에 반환할 수 있음
- 계약자는 마지막 버튼을 5초 유지함

### 공동 발진

- 원자로, 항법, 화물 고정 설비 중 두 개가 작동하면 게이지 증가
- 세 개 모두 작동하면 빠르게 증가
- 한 개 이하이면 감소
- 설비 조작 후 6초간 자동 유지
- 계약자 이탈 후에도 충성 플레이어 두 명이 발진 유지, 한 명이 재보정 또는 추격 가능

### 종료

- 공동 게이지 선행 완료: 공동 성공, 계약 실패
- 마지막 버튼 선행 완료: 계약 성공 즉시 스테이지 종료
- 시간 초과: 전체 실패
- 전원은 다음 스테이지에 즉시 참여함

## 9. 코미디 수리

### 냉각수 호스

- 한 명이 호스를 잡고 한 명이 밸브를 잠금
- 놓치면 제한 범위 물줄기와 미끄러운 바닥
- 진행 일부 유지
- 정상·완벽 판정 공동 지급

### 압력 밸브

- 한 명은 압력 숫자, 한 명은 밸브 조작
- 과압 시 1–3초 증기 충격과 물건 드롭
- 목표 구간 유지로 정상·완벽 판정

수리는 10–20초, 실패해도 전체 초기화 없음, 물리 사고는 짧고 회복 가능하게 제한함. 초기 두 작업의 반복 피로는 계약자 없는 판에서 별도 측정함.

## 10. DDD 리듬

```text
협동 수리 성공과 짧은 사고
→ 개인 할당량 상승
→ 합법 업무 진행
→ 계약 유혹 또는 계약 경보
→ 전원 체크섬과 거짓 기호
→ 실제 후보와 무고한 후보 공개
→ 두 주장 중 하나를 선택해 재보정
→ 키카드 추격과 공동 발진
→ 마지막 버튼 니어미스
→ 계약자 공개와 할당량 정산
```

## 11. 초기 검증 범위

- 4인, 고정 맵 1개, 3스테이지
- 3.5B 개인 할당량
- 합법 업무 4종
- 블랙 계약 1종
- 전원 체크섬과 2인 후보
- 재보정 1회
- 냉각수 호스와 압력 밸브
- 2-of-3 공동 발진
- 키카드, 포드, 돌진과 마지막 버튼
- Discord 음성

6인, 공개 매칭, 자체 음성, 몬스터, 모듈 맵, CCTV, 리플레이, 전용 서버와 장기 능력 성장은 제외함.

## 12. 적대 검증 지표

- 후보 생성 후 시스템 정보만으로 계약자가 자동 확정되지 않음
- 무고한 후보가 자신에게 대응 기회가 없었다고 느끼는 비율 25% 미만
- 계약자가 자신의 기호와 행동에 대해 거짓 설명한 판 90% 이상
- 패널 재보정 선택 정확도 40–75%
- 계약 수락률 20–65%, 성공률 30–50%
- 계약자가 없는 판 재플레이 의향 60% 이상
- 충성과 계약 재미 차이 1점 이내
- 고의 팀 방해가 개인적으로 유리하다고 답한 비율 10% 미만
- 3스테이지 재대결 의향 60% 이상

## 13. 폐기 기준

- 두 후보 선택이 사회적 판단이 아니라 동전 던지기로 느껴짐
- 입력 속도 하나가 항상 정답을 알려줌
- 무고한 후보가 임의로 희생되는 느낌을 받음
- 계약자가 거짓 설명 없이 성공함
- 할당량 때문에 계약이 항상 거절되거나 항상 수락됨
- 계약 없는 판 수리가 재미없음
- 계약 성공이 트롤링으로 평가됨
- 3스테이지 후 재플레이 의향이 절반 미만임

## 14. 프로토타입 상태

현재 Unity 프로토타입은 v0.2 규칙을 구현함. 반복 검증 종료 전까지 v0.9 규칙은 코드에 반영하지 않음.
